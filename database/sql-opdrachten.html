Niveau 1 INNER JOIN & AS
01
INNER JOINStudenten met hun klas
Toon alle studenten met de naam van hun klas. Gebruik AS om de twee naam-kolommen te onderscheiden: student_naam en klas_naam.
▼
📌 Uitleg Een JOIN koppelt twee tabellen op basis van een gemeenschappelijke waarde. De basisstructuur is:
SELECT ... FROM tabel1 JOIN tabel2 ON tabel1.kolom = tabel2.kolom

Beide tabellen hebben een kolom naam. Om verwarring te voorkomen gebruik je AS om een kolom een andere naam te geven in het resultaat:
studenten.naam AS student_naam
SELECT studenten.naam AS student_naam, klassen.naam AS klas_naam FROM studenten JOIN klassen ON studenten.klas_id = klassen.id;
SELECT studenten.naam AS student_naam, klassen.naam AS klas_naam
FROM studenten
JOIN klassen ON studenten.klas_id = klassen.id;
🔍 Controllevraag
Wat doet een INNER JOIN?
A Toont alle rijen uit beide tabellen, ook als er geen overeenkomst is
B Toont alleen rijen die in beide tabellen een overeenkomst hebben
C Toont alle rijen uit de eerste tabel, met NULL voor ontbrekende matches
D Combineert alle kolommen van beide tabellen zonder voorwaarde
✅ Correct! Een INNER JOIN geeft alleen de rijen terug waarbij er een overeenkomst is in beide tabellen. Studenten zonder een klas_id (of met een klas_id die niet bestaat) verschijnen dus niet in het resultaat.
02
INNER JOINStudenten met klas en lokaal
Toon de naam van elke student (student_naam), de naam van zijn klas (klas_naam) en het lokaal. Sorteer op klas_naam.
▼
📌 Uitleg Na de JOIN heb je toegang tot alle kolommen van beide tabellen. Kolommen uit klassen (zoals lokaal) hoef je niet van een alias te voorzien als ze uniek zijn. Alleen als twee tabellen dezelfde kolomnaam hebben, is een alias of tabelnaam verplicht.
SELECT studenten.naam AS student_naam, klassen.naam AS klas_naam, klassen.lokaal FROM studenten JOIN klassen ON studenten.klas_id = klassen.id ORDER BY klas_naam;
SELECT studenten.naam AS student_naam,
       klassen.naam AS klas_naam,
       klassen.lokaal
FROM studenten
JOIN klassen ON studenten.klas_id = klassen.id
ORDER BY klas_naam;
🔍 Controllevraag
Waar schrijf je de koppelingsvoorwaarde bij een JOIN?
A In de WHERE-clausule, net als andere filters
B Direct na de naam van de tweede tabel, na het keyword ON
C In de SELECT-clausule, tussen de kolomnamen
D Na ORDER BY, aan het einde van de query
✅ Juist! De koppeling staat altijd direct na de tweede tabelnaam: JOIN klassen ON studenten.klas_id = klassen.id. Je kunt extra filters nog altijd toevoegen via WHERE, maar de JOIN-conditie hoort bij ON.
03
INNER JOINStudenten in klas 1A met hun mentor
Toon de naam van alle studenten die in klas '1A' zitten, samen met de naam van hun mentor. Gebruik een INNER JOIN en filter met WHERE.
▼
📌 Uitleg Na een JOIN kun je gewoon een WHERE toevoegen om te filteren. Je kunt filteren op kolommen van beide tabellen. Hier filter je op klassen.naam = '1A' omdat je de klasnaam wilt controleren, niet de student.
SELECT studenten.naam AS student_naam, klassen.mentor FROM studenten JOIN klassen ON studenten.klas_id = klassen.id WHERE klassen.naam = '1A';
SELECT studenten.naam AS student_naam, klassen.mentor
FROM studenten
JOIN klassen ON studenten.klas_id = klassen.id
WHERE klassen.naam = '1A';
🔍 Controllevraag
Wat gebeurt er bij een INNER JOIN met een student die klas_id = NULL heeft?
A De student verschijnt met NULL als klas_naam in het resultaat
B De student verschijnt helemaal niet in het resultaat
C SQL geeft een foutmelding
D De student wordt gekoppeld aan de eerste klas in de tabel
✅ Juist! Bij een INNER JOIN worden alleen rijen getoond waarbij er een match is. NULL kan nooit matchen met een id, dus studenten zonder klas worden uitgesloten. Wil je ze wél zien? Gebruik dan een LEFT JOIN (volgende opdrachten).
Niveau 2 LEFT JOIN
04
LEFT JOINAlle studenten, ook zonder klas
Toon alle studenten, ook de studenten die nog niet zijn ingedeeld in een klas. Gebruik een LEFT JOIN. Toon student_naam en klas_naam.
▼
📌 Uitleg Een LEFT JOIN toont alle rijen uit de linker tabel (de tabel na FROM), ook als er geen overeenkomst is in de rechter tabel. Kolommen uit de rechter tabel zijn dan NULL.

In de testdata zijn drie studenten (Lukas, Soraya, Milan) zonder klas. Die zijn bij een INNER JOIN onzichtbaar, maar bij een LEFT JOIN verschijnen ze met NULL als klas_naam.
SELECT studenten.naam AS student_naam, klassen.naam AS klas_naam FROM studenten LEFT JOIN klassen ON studenten.klas_id = klassen.id;
SELECT studenten.naam AS student_naam, klassen.naam AS klas_naam
FROM studenten
LEFT JOIN klassen ON studenten.klas_id = klassen.id;
🔍 Controllevraag
Wat staat er in de kolom klas_naam voor een student die geen klas heeft bij een LEFT JOIN?
A Een lege string ""
B De waarde 0
C NULL — er is geen waarde
D De tekst 'geen klas'
✅ Correct! Bij een LEFT JOIN verschijnen niet-gematchte rijen met NULL in alle kolommen van de rechter tabel. Een lege string en 0 zijn echte waarden — NULL betekent "onbekend" of "niet aanwezig".
05
LEFT JOINAlleen studenten zonder klas
Toon uitsluitend de studenten die geen klas hebben. Gebruik een LEFT JOIN gecombineerd met IS NULL.
▼
📌 Uitleg Door na een LEFT JOIN te filteren op WHERE klassen.id IS NULL houd je alleen de rijen over die geen match hadden. Dit is een veelgebruikt patroon om te zoeken naar "ontbrekende" koppelingen.

Let op: schrijf altijd IS NULL — niet = NULL. In SQL werkt = NULL niet zoals verwacht.
SELECT studenten.naam AS student_naam FROM studenten LEFT JOIN klassen ON studenten.klas_id = klassen.id WHERE klassen.id IS NULL;
SELECT studenten.naam AS student_naam
FROM studenten
LEFT JOIN klassen ON studenten.klas_id = klassen.id
WHERE klassen.id IS NULL;
🔍 Controllevraag
Waarom gebruik je IS NULL in plaats van = NULL?
A Er is geen verschil, beide werken hetzelfde
B In SQL werkt = NULL niet — vergelijking met NULL geeft altijd UNKNOWN terug
C IS NULL werkt alleen na een LEFT JOIN
D = NULL controleert de waarde, IS NULL controleert het datatype
✅ Juist! NULL is geen gewone waarde maar een toestand ("onbekend"). De expressie klassen.id = NULL levert altijd UNKNOWN op — ook als de waarde echt NULL is — en de rij wordt dus niet geselecteerd. Gebruik altijd IS NULL of IS NOT NULL.
Niveau 3 Meerdere JOINs achter elkaar
06
3× JOINWelke vakken volgt elke student?
Toon welke vakken elke student volgt. Koppel via 3 JOINs: studenten → klassen → klas_vakken → vakken. Toon student_naam, klas_naam en vak_naam. Sorteer op student_naam.
▼
📌 Uitleg Je kunt meerdere JOINs achter elkaar schrijven. Elke JOIN bouwt voort op het resultaat van de vorige. De koppelketen hier is:

studenten → (klas_id) → klassen → (id) → klas_vakken → (vak_id) → vakken

Structuur: FROM studenten JOIN klassen ON ... JOIN klas_vakken ON ... JOIN vakken ON ...
SELECT studenten.naam AS student_naam, klassen.naam AS klas_naam, vakken.naam AS vak_naam FROM studenten JOIN klassen ON studenten.klas_id = klassen.id JOIN klas_vakken ON klassen.id = klas_vakken.klas_id JOIN vakken ON klas_vakken.vak_id = vakken.id ORDER BY student_naam;
SELECT studenten.naam AS student_naam,
       klassen.naam   AS klas_naam,
       vakken.naam    AS vak_naam
FROM studenten
JOIN klassen     ON studenten.klas_id   = klassen.id
JOIN klas_vakken ON klassen.id         = klas_vakken.klas_id
JOIN vakken      ON klas_vakken.vak_id  = vakken.id
ORDER BY student_naam;
🔍 Controllevraag
Hoe voeg je een derde (of vierde) tabel toe aan een JOIN-query?
A Je kunt maximaal twee tabellen joinen in één query
B Door een extra JOIN tabel ON ... toe te voegen onder de vorige JOIN
C Door de tabelnamen met komma's te scheiden in de FROM-clausule
D Door gebruik te maken van UNION
✅ Precies! Je kunt zoveel JOINs achter elkaar schrijven als je wilt. Elke JOIN breidt het resultaat uit met een extra tabel. Komma's in FROM (een cross join) is technisch mogelijk maar verouderd en gevaarlijk — gebruik altijd de expliciete JOIN-syntax.
